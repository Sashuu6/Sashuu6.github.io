Setup Localhost On Your Mac

Macintosh As A Localhost(supports Mohave too)
  • 11 Oct 2018

I first started developing commerial products using HTML, CSS, Javascript and many more languages that were used to code client-side of a web application. I first encountered server-side languages while having a technical conversation with one of my seniors. This conversation motivated to learn PHP. While surfing the internet, I found we couldn't execute server-side languages like client-side. That is, I couldn't see the output just by executing the code via a browser. To execute server-side languages like PHP, I had to establish a server inside my mac.

The following are the steps I used to setup a localhost on my mac.

  1. On mac, we dont have to install a server, it's already pre-installed. We just have to enable it.
  2. sudo apachectl start
  3. To check whether the server is not running or not, go to http://localhost on the mac's browser.
  4. PHP is also preinstalled on mac, we just have to enable it.
  5. First, we have to edit the server's config file.
  6. sudo nano /etc/apache2/httpd.conf
  7. Now, we will search for PHP in this file using ctrl+w
  8. Remove # from the below line.
  9. LoadModule php7_module libexec/apache2/libphp7.so
  10. Restart apache server
  11. sudo apachectl restart
  12. TWe have a create a folder named "Sites" on the home directory.
  13. sudo mkdir /Users/<--your_username-->/Sites
  14. Lets save a file named index.php with the following contents.
  15.  <?php echo "Hello From Sites Folder!”; phpinfo();?>
  16. We have to add the Sites fold on the config file. Edit the file using the following command.
  17. sudo nano /etc/apache2/httpd.conf
  18. Inside the file, search for the word "library" using ctrl+w.
  19. Replace /Library/WebServer/Documents with /Users/<--your_username-->/Sites
  20. Now, we need a database and a DBMS for the localhost. Lets download mysql for mac.
  21. First we will download MySQL from this link.
  22. Install MySQL from dmg file.
  23. Now, we will configure MqSQL by opening MySQL from system preferences. Add a password and click legacy password. After this step, just click "start MySQL server".
  24. You can use sql through bash by executing the following command.
  25. sudo /usr/local/mysql/bin/mysql -u root -p
  26. Or you can use Sequel pro. Sequel pro is an application that can be used to any database remotely. That is, it can be used to connect to localhost and anyother server established on the network.
  27. Now, we will make the server communication encrypted. That is, we will enable SSL/HTTPS on localhost. Execute the following command to edit the config file.
  28. sudo nano /etc/apache2/httpd.conf
  29. Search for socache_shmcb_module using ctrl+w.
  30. Remove # from the following lines 
  31. socache_shmcb_module libexec/apache2/mod_socache_shmcb.so

  32. serach for LoadModule ssl_module libexec/apache2/mod_ssl.so and remove # from it.

  33. LoadModule ssl_module libexec/apache2/mod_ssl.so

  34. Delete # from Include /private/etc/apache2/extra/httpd-ssl.conf

  35. /private/etc/apache2/extra/httpd-ssl.conf

  36. We have to replace www.example.com:443 with localhost from the below file.

  37. sudo nano /etc/apache2/extra/httpd-ssl.conf

  38. Replace Library/WebServer/Documents. Replace that with /Users/<--your_username-->/Sites

  39. Add the following lines below <VirtualHost_default_:443>

  40. <Directory "/Users/<--your_username-->/Sites"> 
    Options All 
    MultiviewsMatch Any 
    AllowOverride All 
    Require all granted 
    </Directory>

  41. Open the following config file.

  42. sudo nano /etc/ssl/openssl.cnf

  43. Add the following lines at the end.

  44. [ san ] 
    subjectAltName                  = DNS:localhost
  45. Now we generate a key using openSSL.
  46. sudo openssl req -extensions san -config /etc/ssl/openssl.cnf -x509 -nodes -newkey rsa:4096 -keyout /private/etc/apache2/server.key -out /private/etc/apache2/server.crt -days 365 -subj "/C=your_country/ST=your_state/L=your_city/O=hostname/CN=localhost"

  47. Now we will add the certificate using the following command.

  48. sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain /private/etc/apache2/server.crt

  49. Now just restart the apache server.

  50. sudo apachectl restart

  51. We have not successfully established a server inside Mac
